Refinements & Flexibility

Features (3)

params Collections

Syntax

params now works with any collection type — Span<T>, List<T>, IEnumerable<T>, not just arrays.

Why it was added

params T[] always allocated an array. With params Span<T>, the compiler can stack-allocate, avoiding heap allocation for small argument lists.

C# Example

// params with Span<T> — zero allocation for small args
public int Sum(params ReadOnlySpan<int> values)
{
    int total = 0;
    foreach (var v in values)
        total += v;
    return total;
}

// params with any collection type
public void Log(params IEnumerable<string> messages)
{
    foreach (var msg in messages)
        Console.WriteLine(msg);
}

// Caller syntax is unchanged
var result = Sum(1, 2, 3, 4, 5);
Log("Hello", "World");

// Can also pass collections directly
int[] numbers = [1, 2, 3];
var total = Sum(numbers);
*

Tip for Java developers

params Span<T> can avoid heap allocation entirely for small argument counts. Java varargs always allocate an array on the heap.

New Lock Type

Performance

System.Threading.Lock provides a faster, more efficient alternative to lock(object).

Why it was added

The traditional lock(obj) uses Monitor which has overhead. The new Lock type is purpose-built for mutual exclusion with better performance.

C# Example

// New Lock type — more efficient than lock(object)
private readonly Lock _lock = new();

public void ThreadSafeMethod()
{
    lock (_lock) // compiler recognizes Lock type
    {
        // critical section
        sharedResource.Update();
    }
}

// Or with explicit scope
public void AlternativeUsage()
{
    using (_lock.EnterScope())
    {
        // critical section
    }
}
*

Tip for Java developers

The new Lock type works with the existing lock keyword. The compiler detects the Lock type and generates optimized code.

Partial Properties

Syntax

Properties can now be partial — declare in one part, implement in another. Great for source generators.

Why it was added

Source generators (like regex, JSON serialization) needed to generate property implementations. Partial properties enable this cleanly.

C# Example

// Partial class with partial property declaration
public partial class UserModel
{
    public partial string Name { get; set; }
    public partial int Age { get; set; }
}

// Implementation (e.g., generated by a source generator)
public partial class UserModel
{
    private string _name = "";
    public partial string Name
    {
        get => _name;
        set
        {
            _name = value;
            OnPropertyChanged(nameof(Name));
        }
    }

    private int _age;
    public partial int Age
    {
        get => _age;
        set
        {
            _age = value;
            OnPropertyChanged(nameof(Age));
        }
    }
}
*

Tip for Java developers

Partial properties extend C#'s source generator story. Combined with partial methods (C# 3.0) and partial classes, they enable powerful code generation.
